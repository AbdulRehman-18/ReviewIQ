# Frontend Prompt вЂ” Customer Review Intelligence Dashboard

## Role
You are a senior frontend engineer building a data-dense, judge-impressing dashboard for a 36-hour hackathon. Your bar: the UI should look like a product a real FMCG analyst would use on Monday morning вЂ” not a generic bootstrap template. Aesthetic reference: Linear + Mixpanel hybrid, dark-mode default, sharp typography, information over decoration.

## Objective
Build a React SPA that consumes the FastAPI backend (contract below) to let non-technical product and marketing teams upload reviews, explore feature-level sentiment, watch trends emerge, triage spam/sarcastic reviews, and export reports.

## Tech Stack (non-negotiable)
- **Framework**: React 18 + Vite + TypeScript
- **Styling**: Tailwind CSS v3 + shadcn/ui components (install via their CLI, not a wrapper library)
- **Charts**: Recharts (preferred вЂ” integrates cleanly with shadcn). Use a single chart component wrapper so theming is consistent.
- **State/Data**: TanStack Query (React Query) v5 for all server state; Zustand for the small amount of client-only state (active product, filters, dark mode)
- **Routing**: React Router v6
- **Forms/Upload**: react-hook-form + zod; react-dropzone for file drop
- **HTTP**: axios with a single `apiClient` instance
- **Icons**: lucide-react
- **PDF/CSV download**: trigger browser download from backend blob вЂ” do not rebuild the PDF client-side

## Design System (set this up first, before any screen)
- **Theme**: dark mode default, light mode toggle. Use shadcn's CSS variable approach. Accent color: a single confident hue вЂ” pick `#6366f1` (indigo-500) or `#10b981` (emerald-500), not both.
- **Typography**: Inter for UI, JetBrains Mono for numbers/IDs/code. Tabular numerals (`font-variant-numeric: tabular-nums`) on every number in charts and tables вЂ” this is non-negotiable for a data dashboard.
- **Spacing**: generous. 24px base gutter, never less than 16px between data regions.
- **Sentiment color language** (use consistently everywhere): positive = `emerald-500`, negative = `rose-500`, neutral = `zinc-400`, ambiguous/sarcastic = `amber-500`, spam = `zinc-600` with diagonal-stripe background.
- **Density**: prefer compact tables with sortable headers over card grids for review lists. Cards are for top-level metrics only.
- **Motion**: subtle only. Framer Motion for page transitions and the trend timeline anomaly pulse. No bouncy spring animations on data.

## Backend Contract (consume exactly вЂ” do not invent endpoints)
Base URL: `http://localhost:8000` (configurable via `VITE_API_BASE_URL`)

- `GET /api/products` в†’ product list with summary stats
- `POST /api/products` в†’ create product
- `POST /api/reviews/upload` (multipart) в†’ ingestion summary
- `POST /api/reviews/stream` в†’ single review analyzed
- `POST /api/reviews/manual` в†’ single review analyzed
- `GET /api/products/{id}/overview` в†’ top-level metrics
- `GET /api/products/{id}/features` в†’ per-feature aggregates
- `GET /api/products/{id}/reviews?filter=&page=&size=` в†’ paginated reviews
- `GET /api/products/compare?ids=1,2,3` в†’ cross-product matrix
- `GET /api/products/{id}/trends?feature=&window=50` в†’ trend snapshots
- `GET /api/products/{id}/trends/emerging` в†’ currently flagged emerging issues
- `GET /api/products/{id}/trends/anomalies` в†’ anomaly timeline
- `GET /api/reports/{product_id}/pdf` в†’ blob download
- `GET /api/reports/{product_id}/json` в†’ structured export

Define TypeScript types in `src/types/api.ts` that mirror the backend exactly. Use TanStack Query for all reads with sensible `staleTime` (60s for aggregates, 0 for live-demo mode).

## Critical UX Details
- **Loading**: never show spinners on the whole page. Use shadcn's `Skeleton` components that match the shape of the component about to appear.
- **Empty states**: every list/chart needs a `EmptyState` with an icon, one-line explanation, and a primary CTA ("Upload reviews" / "Select a product").
- **Errors**: inline, not toast-only. If a chart fails, show the error *inside* the `ChartFrame` with a retry button.
- **Numbers**: always formatted. `1,234` not `1234`. `38%` not `0.38`. Percent deltas show sign and color (`+12pp` rose / `в€’12pp` emerald).
- **Dates**: relative for recent (`2h ago`), absolute for older (`Apr 14`). Use `date-fns`.
- **Accessibility**: semantic HTML, focus-visible styles, aria-labels on icon-only buttons, keyboard-navigable tables.
- **Responsive**: desktop-first (judges will see a laptop), but must not break below 1024px. Mobile is not a goal.
